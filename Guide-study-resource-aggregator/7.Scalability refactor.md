# Why Refactor?

Imagine your project grows and you add:

* Difficulty Levels
* Caching
* Search History
* Multiple LLM Providers
* PDF Export
* Web Interface

Your single file might become:

```text
main.py
↓
500+ lines
```

Finding bugs and adding features becomes harder.

Instead, we separate responsibilities.

# books_service.py

```python
import requests


def fetch_books(topic):

    url = (
        f"https://openlibrary.org/"
        f"search.json?q={topic}"
    )

    response = requests.get(url)

    data = response.json()

    books = []

    for book in data["docs"][:5]:

        books.append(
            {
                "title": book.get("title"),
                "author": book.get(
                    "author_name",
                    ["Unknown"]
                )[0],
            }
        )

    return books
```

# youtube_service.py

```python
import os
import requests

YOUTUBE_API_KEY = os.getenv(
    "YOUTUBE_API_KEY"
)


def fetch_youtube_videos(topic):

    url = (
        "https://www.googleapis.com/"
        "youtube/v3/search"
    )

    params = {
        "part": "snippet",
        "q": topic,
        "type": "video",
        "maxResults": 5,
        "key": YOUTUBE_API_KEY,
    }

    response = requests.get(
        url,
        params=params,
    )

    videos = []

    for item in response.json()["items"]:

        videos.append(
            {
                "title":
                    item["snippet"]["title"],
                "channel":
                    item["snippet"][
                        "channelTitle"
                    ],
                "url":
                    "https://youtube.com/watch?v="
                    + item["id"]["videoId"],
            }
        )

    return videos
```

# roadmap_prompt.py

```python
def build_prompt(
    topic,
    days,
    videos,
    books,
):

    video_text = "\n".join(
        [
            f"- {video['title']}"
            for video in videos
        ]
    )

    book_text = "\n".join(
        [
            f"- {book['title']}"
            for book in books
        ]
    )

    return f"""
You are a learning assistant.

Topic: {topic}

Days: {days}

Videos:
{video_text}

Books:
{book_text}

Create a beginner-friendly
day-wise roadmap.

Return only the roadmap.
"""
```

# llm_service.py

```python
import os
import requests

OPENROUTER_API_KEY = os.getenv(
    "OPENROUTER_API_KEY"
)


def generate_roadmap(prompt):

    url = (
        "https://openrouter.ai/api/v1/"
        "chat/completions"
    )

    headers = {
        "Authorization":
            f"Bearer {OPENROUTER_API_KEY}",
        "Content-Type":
            "application/json",
    }

    payload = {
        "model":
            "openai/gpt-oss-20b",
        "messages": [
            {
                "role": "user",
                "content": prompt,
            }
        ],
    }

    response = requests.post(
        url,
        headers=headers,
        json=payload,
    )

    return (
        response.json()["choices"][0]
        ["message"]["content"]
    )
```

# main.py

Notice how clean the application becomes.

```python
from dotenv import load_dotenv

from services.books_service import (
    fetch_books,
)

from services.youtube_service import (
    fetch_youtube_videos,
)

from services.llm_service import (
    generate_roadmap,
)

from prompts.roadmap_prompt import (
    build_prompt,
)

load_dotenv()


def main():

    topic = input(
        "Enter topic: "
    )

    days = input(
        "Enter number of days: "
    )

    books = fetch_books(topic)

    videos = fetch_youtube_videos(
        topic
    )

    prompt = build_prompt(
        topic,
        days,
        videos,
        books,
    )

    roadmap = generate_roadmap(
        prompt
    )

    print(roadmap)


if __name__ == "__main__":
    main()
```

# Benefits of This Structure

### Easier Debugging

If book fetching breaks:

```text
services/books_service.py
```

is the first place to check.

---

### Easier Testing

Each service can be tested independently.

Example:

```python
fetch_books("python")
```

without running the entire application.

---

### Easier Feature Development

Want to add caching?

Create:

```text
services/cache_service.py
```

Want to add Gemini?

Create:

```text
services/gemini_service.py
```

No need to touch unrelated files.

---

### Easier Team Collaboration

One person can work on:

```text
books_service.py
```

while another works on:

```text
llm_service.py
```

without conflicts.

# Industry Perspective

Most professional applications follow a similar philosophy:

```text
Controllers
Services
Repositories
Utilities
Configurations
```

The exact names differ, but the idea remains the same:

> One file should have one primary responsibility.

This refactor introduces that mindset without adding too much complexity.
