# Part 1: Understanding the Project

## Overview

In this project, you will build a **Study Resource Aggregator + AI Roadmap Generator**.

The application will:

1. Accept a topic from the user.
2. Accept the number of days they want to learn.
3. Search for learning resources from multiple sources.
4. Organize those resources into a structured format.
5. Send the resources to an AI model.
6. Generate a day-wise learning roadmap.

### Example

**Input**

```text
Topic: Machine Learning
Days: 7
```

**Output**

```text
Day 1
- Introduction to Machine Learning
- Watch: ML Crash Course
- Read: Introduction Chapters

Day 2
- Supervised Learning
- Watch: Regression Explained
- Read: Learning ML Chapter 2

...
```

---

# What Are We Actually Building?

Most students struggle with learning because resources are scattered across different platforms.

For example, if someone wants to learn React, they might need to:

* Search YouTube for tutorials
* Search Google for books
* Search blogs for documentation
* Figure out what order to learn things in

Our application automates this process.

Instead of manually searching, the application will:

1. Find relevant resources
2. Collect them into one place
3. Ask an AI model to create a learning plan

# Breaking Down The Workflow

## Step 1: User Input

The user provides:

```text
Topic: React
Days: 14
```

This becomes the starting point of the application.

---

## Step 2: Fetch Learning Resources

The application will query external services.

### YouTube

We will search for:

```text
React Tutorials
```

and collect:

* Video titles
* Channel names
* Video URLs

---

### Open Library

We will search for:

```text
React
```

and collect:

* Book titles
* Author names
* Book information

---

## Step 3: Aggregate Data

Aggregation means combining information from multiple sources into a single structure.

Example:

```json
{
  "videos": [
    {
      "title": "React Crash Course",
      "channel": "Traversy Media"
    }
  ],
  "books": [
    {
      "title": "Learning React",
      "author": "Alex Banks"
    }
  ]
}
```

Instead of dealing with multiple APIs separately, we now have one clean data structure.

---

## Step 4: Generate The Roadmap

After collecting resources, we send them to an LLM.

The model receives:

* Topic
* Number of days
* Videos
* Books

Example:

```text
Topic: React
Days: 14

Videos:
- React Crash Course
- React Hooks Explained

Books:
- Learning React
- React Design Patterns
```

The AI then creates a structured learning roadmap.

# Important Concepts

## What Is An API?

API stands for Application Programming Interface.

Think of an API as a waiter in a restaurant.

```text
You
 ↓
Waiter
 ↓
Kitchen
```

You tell the waiter what you want.

The waiter delivers the request.

The kitchen prepares the response.

The waiter returns the result.

Similarly:

```text
Your Program
 ↓
API
 ↓
Server
```

Your application sends a request.

The API returns data.

---

## What Is JSON?

Most APIs return data in JSON format.

Example:

```json
{
  "title": "Learning React",
  "author": "Alex Banks"
}
```

Python automatically converts this into a dictionary.

Example:

```python
book["title"]
```

Output:

```text
Learning React
```

JSON is the most common format used when working with APIs.

---

## What Is An LLM?

LLM stands for Large Language Model.

Examples include:

* GPT models
* Gemini
* Claude
* Llama

An LLM can:

* Answer questions
* Summarize content
* Generate plans
* Write code
* Analyze data

In this project, we use an LLM as a roadmap generator.
