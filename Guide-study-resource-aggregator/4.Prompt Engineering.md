# What Is Prompt Engineering?

Prompt Engineering is the process of designing instructions for an AI model.

Think of it this way:

Poor instructions produce poor results.

Good instructions produce good results.

---

## Bad Prompt

```text
Teach me React.
```

Problems:

* No structure
* No timeline
* No resources
* No guidance

---

## Better Prompt

```text
Create a 14-day learning roadmap for React.

Include:
- Topics to study
- Videos to watch
- Books to read
```

Now the model has more direction.

---

## Great Prompt

```text
Create a beginner-friendly 14-day React roadmap.

Use the provided videos and books.

For each day include:
- Topics
- Videos
- Reading material

Return only the roadmap.
```

Much better.

---

# Why Provide Context?

Imagine asking:

```text
Create a Machine Learning roadmap.
```

The model uses only its training knowledge.

Instead, we can provide real resources.

Example:

```text
Videos:
- ML Crash Course
- Neural Networks Explained

Books:
- Hands-On Machine Learning
- Introduction to Machine Learning
```

Now the model can generate a roadmap based on actual resources.

This technique is often called:

## Retrieval-Augmented Generation (RAG)

Although this project is a simplified version, the idea is similar:

```text
External Data
      +
LLM
      =
Better Output
```

# Building the Prompt

Let's assume we already have:

```python
videos = [
    {
        "title": "ML Crash Course",
        "url": "..."
    }
]

books = [
    {
        "title": "Hands-On Machine Learning",
        "author": "Aurelien Geron"
    }
]
```

The LLM cannot understand Python dictionaries directly.

We need to convert them into readable text.

---

# Formatting Video Data

Example:

```python
video_text = "\n".join(
    [
        f"- {video['title']}"
        for video in videos
    ]
)
```

Output:

```text
- ML Crash Course
- Neural Networks Explained
- Deep Learning Basics
```

---

# Formatting Book Data

```python
book_text = "\n".join(
    [
        f"- {book['title']} by {book['author']}"
        for book in books
    ]
)
```

Output:

```text
- Hands-On Machine Learning by Aurelien Geron
- Introduction to Machine Learning by Ethem Alpaydin
```

# Giving Instructions to the Model

Now we add clear instructions.

```python
prompt = f"""
You are a learning assistant.

Topic: {topic}

Total Days: {days}

Videos:
{video_text}

Books:
{book_text}

Task:

1. Create a day-wise roadmap.
2. Use the provided resources.
3. Keep the roadmap beginner friendly.
4. Include topics, videos, and books.
5. Return only the roadmap.
"""
```

Notice how explicit instructions make the task easier for the model.

# Understanding Chat Completions

Modern LLM APIs usually expect messages.

Example:

```json
{
  "messages": [
    {
      "role": "user",
      "content": "Hello"
    }
  ]
}
```

Think of this as a conversation.

---

# Sending the Prompt

Example request structure:

```python
payload = {
    "model": "openai/gpt-oss-20b",
    "messages": [
        {
            "role": "user",
            "content": prompt
        }
    ]
}
```

This tells the model:

```text
Use this prompt
Generate a response
```

---

# Understanding Temperature

Many models allow:

```python
"temperature": 0.7
```

Lower values:

```text
0.1
```

* More predictable
* More consistent

Higher values:

```text
1.0
```

* More creative
* More random

For roadmaps:

```text
0.3 - 0.7
```

works well.

---

# Understanding the Response

A typical response looks like:

```json
{
  "choices": [
    {
      "message": {
        "content": "Day 1..."
      }
    }
  ]
}
```

To access the roadmap:

```python
roadmap = (
    response.json()
    ["choices"][0]
    ["message"]["content"]
)
```

Now we have plain text.

# Common Prompt Engineering Mistakes

## Too Vague

```text
Teach me React.
```

---

## Missing Constraints

```text
Create a roadmap.
```

How many days?

What level?

What resources?

Not specified.

---

## Better

```text
Create a beginner-friendly
14-day React roadmap using
the provided books and videos.
```

Specific prompts usually produce better results.
