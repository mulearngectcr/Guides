# 03 - Testing API Key Authentication

> Day 6 – Testing Protected Endpoints

Now that we have added API key authentication to our Todo API, let's learn how to test it.

This is an important step because our endpoints are no longer public.

Every request must include the correct API key.

# Understanding The New Input Field

Open:

```text
GET /todos
```

and click:

```text
Try it out
```

You will notice a new input box:

```text
x-api-key *
```

---

# Why Did This Appear?

Remember this code:

```python
def get_todos(x_api_key: str = Header()):
```

The `Header()` function tells FastAPI:

> This endpoint requires a value from the request headers.

Swagger automatically generates an input field for it.

---

# Testing With The Correct API Key

Suppose your `.env` file contains:

```env
API_KEY=my-secret-key
```

Enter:

```text
my-secret-key
```

inside the:

```text
x-api-key
```

field.

Then click:

```text
Execute
```

---

# Expected Response

If your todo list is empty:

```json
[]
```

If you have added some todos:

```json
[
  {
    "id": 1,
    "title": "Learn FastAPI",
    "checked": false,
    "priority": "medium"
  }
]
```

This means:

> Authentication was successful.

The API key matched, so the server allowed the request.

---

# Testing With An Incorrect API Key

Now try:

```text
wrong-key
```

and click:

```text
Execute
```

---

# Expected Response

```json
{
  "detail": "Invalid API Key"
}
```

You should also see:

```text
401 Unauthorized
```

---

# What Happened?

The server compared:

```text
Received Key:
wrong-key
```

with:

```text
Stored Key:
my-secret-key
```

Since they did not match:

```python
if x_api_key != API_KEY:
```

became:

```python
if "wrong-key" != "my-secret-key":
```

which is:

```python
True
```

So FastAPI raised:

```python
raise HTTPException(
    status_code=401,
    detail="Invalid API Key"
)
```

---

# Testing Without Sending The Header

Let's remove the API key completely.

Leave the input box empty and click:

```text
Execute
```

---

# Expected Response

FastAPI automatically returns:

```json
{
  "detail": [
    {
      "type": "missing",
      "loc": ["header", "x-api-key"],
      "msg": "Field required"
    }
  ]
}
```

with status code:

```text
422 Unprocessable Entity
```

---

# Why 422 Instead Of 401?

A common question:

> Why didn't we get 401 Unauthorized?

Because the request never reached our authentication code.

FastAPI noticed that:

```python
x_api_key: str = Header()
```

requires a header.

Since the header was missing, FastAPI automatically rejected the request before our function executed.

# Complete Testing Flow

A good testing sequence is:

```text
1. POST /todos
   ↓
2. GET /todos
   ↓
3. GET /todos/1
   ↓
4. PUT /todos/1
   ↓
5. DELETE /todos/1
```

Make sure to send the API key every time.

# Visualizing The Request

When you click Execute in Swagger, the request looks something like this:

```http
GET /todos HTTP/1.1
Host: localhost:8000
X-API-Key: my-secret-key
```

FastAPI automatically converts:

```http
X-API-Key
```

into:

```python
x_api_key
```

inside your function.
