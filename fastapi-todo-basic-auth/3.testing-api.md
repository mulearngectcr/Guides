# 03 - Testing API Key Authentication

> Day 6 – Testing Protected Endpoints

Now that we have added API key authentication, let's learn how to test it using Swagger UI.

---

# Starting The Server

```bash
uvicorn main:app --reload
```

---

# Opening Swagger UI

Open:

```text
http://127.0.0.1:8000/docs
```

---

# The Authorize Button

You will now see an **Authorize** 🔓 button at the top right of the page.

This appears automatically because we used `APIKeyHeader`.

FastAPI told Swagger:

> This API requires an API key.

Swagger responded by adding the Authorize button.

---

# Entering Your API Key

Click the **Authorize** button.

A dialog box appears:

```text
X-API-Key
Value: ____________
```

Enter your API key:

```text
my-secret-key
```

Click **Authorize**, then **Close**.

The lock icon changes to 🔒 — this means Swagger will now send your key automatically with every request.

---

# Testing GET /todos

Click:

```text
GET /todos
```

Then:

```text
Try it out → Execute
```

Expected response if your list is empty:

```json
[]
```

You did not need to type your key again — Swagger attached it automatically.

---

# Testing With A Wrong Key

Click **Authorize** again.

Click **Logout**, then enter:

```text
wrong-key
```

Click **Authorize** → **Close**.

Now try:

```text
GET /todos → Execute
```

Expected response:

```json
{
  "detail": "Invalid API Key"
}
```

With status code:

```text
401 Unauthorized
```

---

# Testing With No Key

Click **Authorize** → **Logout** → **Close** without entering anything.

Try any endpoint.

Expected response:

```json
{
  "detail": "Not authenticated"
}
```

With status code:

```text
403 Forbidden
```

FastAPI catches the missing header before your function even runs.

---

# Complete Testing Flow

Re-enter your correct API key first.

Then test in this order:

```text
1. POST /todos        → 201 Created
2. GET /todos         → 200 OK, see your todo
3. GET /todos/1       → 200 OK, see one todo
4. PUT /todos/1       → 200 OK, todo updated
5. DELETE /todos/1    → 200 OK, todo deleted
```

Try each one with a wrong key as well to confirm 401 is returned every time.

---

# What Swagger Is Sending Behind The Scenes

When you click Execute, Swagger sends:

```http
GET /todos HTTP/1.1
Host: localhost:8000
X-API-Key: my-secret-key
```

The `X-API-Key` header is attached automatically because you entered it in the Authorize dialog.

---

You should now understand:

- How to enter your API key once and have it apply to all requests
