# 02 - Building API Key Authentication

> Day 6 – Protecting Our Todo API

In this guide, we will add authentication to our Todo API using a simple API key.

This approach is intentionally kept simple for learning purposes.

Real-world applications typically use:

- JWT Tokens
- Database-stored API Keys
- OAuth2
- Session-based Authentication

But the core idea is exactly the same:

The client sends credentials, and the server checks whether they are valid.

---

# What Are We Building?

Currently, anyone can access our API:

```http
GET /todos
POST /todos
PUT /todos/1
DELETE /todos/1
```

We want to protect these routes.

Only users who know our secret API key should be allowed to use them.

---

# Authentication Flow

```text
Client
│
│ X-API-Key: my-secret-key
│
▼

FastAPI Server
│
│ Compare with stored API key
│
├── Match
│     ↓
│   Allow Request
│
└── No Match
      ↓
   401 Unauthorized
```

```python
import os
```

---

# Understanding the New Imports

---

## APIKeyHeader

```python
from fastapi.security import APIKeyHeader
```

This tells FastAPI:

> This API uses an API key sent through a header.

FastAPI uses this to automatically show the Authorize button in Swagger UI.

---

## Security

```python
from fastapi import Security
```

Used to apply a security scheme to a function parameter.

---

## Depends

```python
from fastapi import Depends
```

Allows us to attach a dependency (like authentication) to an endpoint without putting the auth logic inside the endpoint function itself.

---

# Step 4: Load Environment Variables

```python
load_dotenv()
API_KEY = os.getenv("API_KEY")
```

This reads the API key from your `.env` file.

---

# Step 5: Define the Security Scheme

```python
api_key_scheme = APIKeyHeader(name="X-API-Key")
```

This tells FastAPI:

> Expect an API key inside a header called X-API-Key.

FastAPI automatically registers this and shows the Authorize button in Swagger.

---

# Step 6: Create the authenticate Function

```python
def authenticate(api_key: str = Security(api_key_scheme)):
    if api_key != API_KEY:
        raise HTTPException(
            status_code=401,
            detail="Invalid API Key"
        )
```

---

# Understanding This Function

---

## Security(api_key_scheme)

```python
api_key: str = Security(api_key_scheme)
```

FastAPI reads the `X-API-Key` header and passes its value into `api_key` automatically.

---

## Validation

```python
if api_key != API_KEY:
    raise HTTPException(
        status_code=401,
        detail="Invalid API Key"
    )
```

If the key is wrong, FastAPI immediately returns:

```json
{
  "detail": "Invalid API Key"
}
```

with status code 401.

---

# Step 7: Protecting Endpoints with Depends

```python
@app.get("/todos", dependencies=[Depends(authenticate)])
def get_todos():
    return todos
```

---

# Understanding dependencies=[Depends(authenticate)]

```python
dependencies=[Depends(authenticate)]
```

This tells FastAPI:

> Before running get_todos, run authenticate first.

If authentication fails, the endpoint function never runs.

If authentication passes, the endpoint runs normally.

---

# Why Is This Better Than Putting Auth Inside Each Function?

Without `Depends`:

```python
@app.get("/todos")
def get_todos(x_api_key: str = Header()):
    if x_api_key != API_KEY:
        raise HTTPException(status_code=401, detail="Invalid API Key")
    return todos
```

With `Depends`:

```python
@app.get("/todos", dependencies=[Depends(authenticate)])
def get_todos():
    return todos
```

The endpoint function stays clean.

Authentication logic is written once and reused everywhere.

---

# Step 8: Protecting All Endpoints

```python
@app.get("/todos", dependencies=[Depends(authenticate)])
def get_todos():
    return todos

@app.post("/todos", status_code=201, dependencies=[Depends(authenticate)])
def create_todo(todo: Todo):
    ...

@app.get("/todos/{id}", dependencies=[Depends(authenticate)])
def get_todo(id: int):
    ...

@app.put("/todos/{id}", dependencies=[Depends(authenticate)])
def update_todo(id: int, updated_todo: Todo):
    ...

@app.delete("/todos/{id}", dependencies=[Depends(authenticate)])
def delete_todo(id: int):
    ...
```

Every endpoint now requires a valid API key.
