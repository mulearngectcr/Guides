# 02 - Building API Key Authentication

> Day 6 – Protecting Our Todo API

In this guide, we will add authentication to our Todo API using a simple API key.

This approach is intentionally kept simple for learning purposes.

Real-world applications typically use:

- JWT Tokens
- Database-stored API Keys
- OAuth2
- Session-based Authentication

But the core idea is exactly the same:

> The client sends some credentials, and the server checks whether they are valid.

---

# What Are We Building?

Currently, anyone can access our API:

```http
GET /todos
POST /todos
PUT /todos/1
DELETE /todos/1
```

We want to protect these routes.

Only users who know our secret API key should be allowed to use them.

---

# Authentication Flow

Our application will work like this:

```text
Client
│
│ X-API-Key: my-secret-key
│
▼

FastAPI Server
│
│ Compare with stored API key
│
├── Match
│     ↓
│   Allow Request
│
└── No Match
      ↓
   401 Unauthorized
```

---

# Step 1: Install python-dotenv

We need a package called:

```text
python-dotenv
```

Install it:

```bash
pip install python-dotenv
```

---

# Why Do We Need It?

We want to store our API key inside a `.env` file instead of hardcoding it.

Bad:

```python
API_KEY = "my-secret-key"
```

Better:

```env
API_KEY=my-secret-key
```

This makes it easier to change secrets later.

---

# Step 2: Create a .env File

Create a file called:

```text
.env
```

Inside it:

```env
API_KEY=my-secret-key
```

This file stores our secret API key.

---

# Step 3: Import dotenv and os

Add these imports:

```python
from dotenv import load_dotenv
import os
```

---

# What Does os Do?

The `os` module allows Python to interact with the operating system.

One thing it can do is read environment variables.

Example:

```python
os.getenv("API_KEY")
```

This means:

> Give me the value stored inside the API_KEY environment variable.

---

# Step 4: Load Environment Variables

Add:

```python
load_dotenv()
```

Example:

```python
from dotenv import load_dotenv
import os

load_dotenv()
```

---

# What Does load_dotenv() Do?

It reads the `.env` file and loads its values into Python.

---

Before:

```python
os.getenv("API_KEY")
```

returns:

```python
None
```

---

After:

```python
load_dotenv()
```

it returns:

```python
"my-secret-key"
```

---

# Step 5: Read the API Key

Now we store the key in a variable:

```python
API_KEY = os.getenv("API_KEY")
```

---

# What Happens Here?

Suppose our `.env` file contains:

```env
API_KEY=my-secret-key
```

Then:

```python
API_KEY = os.getenv("API_KEY")
```

becomes:

```python
API_KEY = "my-secret-key"
```

---

# Step 6: Receiving Headers

Let's look at one route:

```python
@app.get("/todos")
def get_todos(x_api_key: str = Header()):

    return todos
```

---

# Understanding Header()

`Header()` tells FastAPI:

> Read a value from the HTTP headers.

---

## Python Code

```python
x_api_key: str = Header()
```

---

## Incoming Request

```http
GET /todos

X-API-Key: my-secret-key
```

---

## Result

FastAPI automatically does:

```python
x_api_key = "my-secret-key"
```

for us.

---

# Automatic Name Conversion

Python variables cannot contain hyphens:

```python
x-api-key
```

is invalid Python.

So FastAPI converts:

```python
x_api_key
```

into:

```http
X-API-Key
```

automatically.

---

# Step 7: Validating The API Key

Now we add our authentication logic:

```python
if x_api_key != API_KEY:

    raise HTTPException(
        status_code=401,
        detail="Invalid API Key"
    )
```

---

# Understanding The Condition

Suppose:

```python
x_api_key = "wrong-key"
```

and:

```python
API_KEY = "my-secret-key"
```

Then:

```python
if x_api_key != API_KEY
```

becomes:

```python
if "wrong-key" != "my-secret-key"
```

which is:

```python
True
```

So FastAPI raises an error.

---

# What Is HTTPException?

FastAPI provides:

```python
HTTPException
```

for returning proper HTTP errors.

Import:

```python
from fastapi import HTTPException
```

---

# Raising The Error

```python
raise HTTPException(
    status_code=401,
    detail="Invalid API Key"
)
```

This immediately stops the function.

Nothing below it runs.

---

# Response Sent To The Client

The user receives:

```json
{
  "detail": "Invalid API Key"
}
```

with status code:

```text
401 Unauthorized
```

---

# Why 401?

```text
401 Unauthorized
```

means:

> You have not provided valid authentication credentials.

This is exactly what happened.

The API key was incorrect.

---

# Protecting Every Route

We need to add:

```python
x_api_key: str = Header()
```

to every route:

```python
@app.get("/todos")
def get_todos(x_api_key: str = Header()):
```

---

```python
@app.post("/todos")
def create_todo(
    todo: Todo,
    x_api_key: str = Header()
):
```

---

```python
@app.put("/todos/{id}")
def update_todo(
    id: int,
    updated_todo: Todo,
    x_api_key: str = Header()
):
```

---

```python
@app.delete("/todos/{id}")
def delete_todo(
    id: int,
    x_api_key: str = Header()
):
```

---

Then, inside each route:

```python
if x_api_key != API_KEY:

    raise HTTPException(
        status_code=401,
        detail="Invalid API Key"
    )
```

---

# Why Are We Repeating Code?

You might think:

> Can't we write this only once?

Yes, absolutely.

FastAPI provides:

- Dependencies
- `Depends()`
- Middleware
- Authentication classes

But these concepts add more complexity.

For learning purposes, repeating the authentication check makes the code easier to understand.

We will improve this later.

---

# Our Final Authentication Logic

The complete process now looks like this:

```text
User
│
│ GET /todos
│ X-API-Key: my-secret-key
│
▼

FastAPI
│
│ Read Header()
│
▼

x_api_key = "my-secret-key"
│
│
Compare with:
API_KEY = "my-secret-key"
│
├── Match
│     ↓
│   Return Todos
│
└── No Match
      ↓
   401 Unauthorized
```
