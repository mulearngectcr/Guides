# 01 - HTTP Headers

> Day 6 – Authentication with API Keys

Before we learn authentication, we need to understand one important concept:

> **HTTP Headers**

Most authentication systems use headers to send credentials from the client to the server.

---

# What Happens When You Open a Website?

Suppose you visit:

```text
https://google.com
```

Your browser sends an HTTP request to Google's servers.

That request contains much more than just:

```http
GET /
```

It also includes extra information about:

- Who sent the request
- What type of data is being sent
- What browser is being used
- Authentication information
- Cookies
- Language preferences

This extra information is called **HTTP Headers**.

---

# Structure of an HTTP Request

A typical HTTP request has three parts:

```text
Request Line
Headers
Body
```

---

## Example Request

```http
GET /todos HTTP/1.1
Host: localhost:8000
User-Agent: Chrome
Content-Type: application/json
X-API-Key: my-secret-key
```

Let's understand each part.

---

# 1. Request Line

The first line:

```http
GET /todos HTTP/1.1
```

contains:

```text
GET
↓
HTTP Method
```

```text
/todos
↓
Endpoint
```

```text
HTTP/1.1
↓
HTTP Version
```

---

# 2. Headers

Everything below the request line is a header:

```http
Host: localhost:8000
User-Agent: Chrome
Content-Type: application/json
X-API-Key: my-secret-key
```

Headers are simply:

> Key-value pairs that provide additional information about the request.

---

# Header Syntax

Headers follow this format:

```text
Key: Value
```

Examples:

```http
Content-Type: application/json

User-Agent: Chrome

Accept-Language: en-US

X-API-Key: my-secret-key
```

---

# Common HTTP Headers

Let's look at some commonly used headers.

---

# Content-Type

Example:

```http
Content-Type: application/json
```

This tells the server:

> The request body contains JSON data.

Without this header, the server may not know how to interpret the data.

---

# User-Agent

Example:

```http
User-Agent: Chrome
```

This tells the server which browser or application made the request.

Examples:

```text
Chrome
Firefox
Postman
curl
Python Requests
```

Many websites use this information for analytics and compatibility.

---

# Authorization

One of the most important headers:

```http
Authorization: Bearer abc123xyz
```

Modern applications often use this header for JWT authentication.

Examples:

```text
GitHub
Google
Discord
Spotify
OpenAI
```

We will learn about JWT tokens later.

---

# X-API-Key

For today's lesson, we will use:

```http
X-API-Key: my-secret-key
```

This sends our API key to the server.

The server checks:

```text
Does the received key match the stored key?
```

If yes:

```text
Access Granted
```

Otherwise:

```text
401 Unauthorized
```

---

# Why Is It Called X-API-Key?

You might wonder:

```text
Why not just API-Key?
```

Historically, custom headers often started with:

```text
X-
```

meaning:

> This is a non-standard or custom header.

Examples:

```text
X-API-Key
X-Request-ID
X-Custom-Token
```

Although modern standards no longer require `X-`, many APIs still use it.

---

# How Authentication Works

Let's see the complete flow.

---

## Step 1: Client Sends Request

```http
GET /todos

X-API-Key: my-secret-key
```

---

## Step 2: Server Receives the Header

The server extracts:

```text
my-secret-key
```

---

## Step 3: Validation

The server compares:

```text
Received Key:
my-secret-key

Stored Key:
my-secret-key
```

---

## Step 4: Decision

If they match:

```text
Access Granted
```

If they don't:

```text
401 Unauthorized
```

---

# Visual Diagram

```text
Client
│

│

Server
│

│
```

# How FastAPI Reads Headers

FastAPI provides a special function called:

```python
Header()
```

Example:

```python
from fastapi import Header


@app.get("/todos")
def get_todos(x_api_key: str = Header()):

    return todos
```

---

# What Does Header() Do?

FastAPI automatically looks for:

```http
X-API-Key: something
```

and stores the value inside:

```python
x_api_key
```

---

# Automatic Conversion

FastAPI converts:

```text
x_api_key
```

into:

```text
X-API-Key
```

automatically.

---

## Python Code

```python
x_api_key: str = Header()
```

---

## HTTP Request

```http
X-API-Key: my-secret-key
```

---

## Result

```python
x_api_key = "my-secret-key"
```

The developer does not need to manually parse the headers.

FastAPI handles everything for us.

---

# What Happens If The Header Is Missing?

Suppose the user sends:

```http
GET /todos
```

without:

```http
X-API-Key
```

FastAPI automatically returns:

```json
{
  "detail": [
    {
      "type": "missing",
      "loc": ["header", "x-api-key"],
      "msg": "Field required"
    }
  ]
}
```

with status code:

```text
422 Unprocessable Entity
```

This means:

> A required header was not provided.
