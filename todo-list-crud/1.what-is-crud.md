# Introduction

Most software applications that we use every day follow a simple pattern:

- We create something.
- We read or view it.
- We update or edit it.
- We delete it.

This pattern is called **CRUD**.

CRUD is one of the most fundamental concepts in backend development and database systems.

Almost every application you use on your phone or computer is built around CRUD operations.

---

# What Does CRUD Mean?

CRUD stands for:

| Letter | Meaning |
| ------ | ------- |
| C      | Create  |
| R      | Read    |
| U      | Update  |
| D      | Delete  |

These four operations represent the basic actions that can be performed on data.

---

# Understanding CRUD with a To-Do App

Imagine a simple to-do application.

You create a task:

```text id="hmyu8g"
Learn FastAPI
```

Later, you open the app to see your tasks.

You decide to mark it as completed.

Finally, after finishing it, you remove it from your list.

These actions are:

```text id="1fjlwm"
Create → Add a new task

Read → View your tasks

Update → Mark the task as completed

Delete → Remove the task
```

That entire application is fundamentally a CRUD application.

# CRUD and APIs

In backend development, users do not directly access databases.

Instead, they communicate through APIs.

The API performs CRUD operations on their behalf.

---

# Example: Creating a Todo

Suppose a user creates a new task.

The application might send:

```http id="3pv4rj"
POST /todos
```

with:

```json id="7ttjlwm"
{
  "title": "Learn FastAPI",
  "checked": false,
  "priority": "medium"
}
```

The server stores this data.

That is a Create operation.

---

# Example: Reading Todos

The user wants to see all tasks.

The application sends:

```http id="9g4jth"
GET /todos
```

The server responds with:

```json id="hv25gh"
[
  {
    "id": 1,
    "title": "Learn FastAPI",
    "checked": false,
    "priority": "medium"
  }
]
```

That is a Read operation.

---

# Example: Updating a Todo

The user finishes learning FastAPI.

The application sends:

```http id="o4wlcs"
PUT /todos/1
```

with:

```json id="n8dy32"
{
  "title": "Learn FastAPI",
  "checked": true,
  "priority": "medium"
}
```

The server updates the existing task.

That is an Update operation.

---

# Example: Deleting a Todo

The user no longer needs the task.

The application sends:

```http id="kg1xk0"
DELETE /todos/1
```

The server removes the todo.

That is a Delete operation.

# CRUD and HTTP Methods

REST APIs typically map CRUD operations to specific HTTP methods.

---

| CRUD Operation | HTTP Method | Example         |
| -------------- | ----------- | --------------- |
| Create         | POST        | POST /todos     |
| Read           | GET         | GET /todos      |
| Update         | PUT         | PUT /todos/1    |
| Delete         | DELETE      | DELETE /todos/1 |

This convention is used by almost every modern web application.

---

# Visualizing CRUD

```text id="vj0mwg"
User
 │
 ├── Create → POST   /todos
 │
 ├── Read   → GET    /todos
 │
 ├── Update → PUT    /todos/1
 │
 └── Delete → DELETE /todos/1
```

The server receives these requests and performs the appropriate operations on the data.

# Why Not Use Only GET?

A beginner might ask:

> Why don't we use GET for everything?

For example:

```text id="tf66mq"
GET /createTodo
GET /deleteTodo
```

The reason is that HTTP methods provide meaning.

When another developer sees:

```http id="pm79nf"
DELETE /todos/1
```

they immediately understand:

> This request removes a todo.

Using standard conventions makes APIs easier to understand and maintain.

---

# REST Conventions

REST stands for:

> Representational State Transfer

It is a style for designing web APIs.

One of its core ideas is:

```text id="z5zjfi"
Use HTTP methods to represent CRUD operations.
```

Examples:

---

Good:

```http id="5dcgnj"
GET    /todos
POST   /todos
PUT    /todos/1
DELETE /todos/1
```

---

Bad:

```http id="md88fh"
GET /getTodos
GET /createTodo
GET /updateTodo
GET /deleteTodo
```

Modern APIs generally follow REST conventions because they are predictable and easy to understand.
