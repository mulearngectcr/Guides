# Introduction

In Day 1, our APIs only returned data.

For example:

```python
@app.get("/")
def home():
    return {
        "message": "Hello from my API"
    }
```

The user requested something, and the server sent back a response.

But now we want users to create their own todos.

For example:

```text
Learn FastAPI
Buy groceries
Finish assignment
```

This raises an important question:

> How does the user send this information to our server?

The answer is:

> Using a Request Body.

---

# What Is a Request Body?

A request body is simply data that a client sends to the server.

Think about filling out a form on a website.

For example:

```text
Title: Learn FastAPI
Priority: Medium
Completed: False
```

When you click:

```text
Submit
```

the browser sends this information to the server.

That data is called the **request body**.

---

# Real-World Example: Instagram

Suppose you create a new Instagram post.

You write:

```text
Beach day with friends!
```

The Instagram app sends something similar to:

```http
POST /posts
```

with:

```json
{
  "caption": "Beach day with friends!"
}
```

The caption is part of the request body.

---

# Real-World Example: Amazon

Imagine buying two books.

The application might send:

```http
POST /orders
```

with:

```json
{
  "product_id": 15,
  "quantity": 2
}
```

Again, this information lives inside the request body.

---

# Real-World Example: YouTube

When posting a comment:

```text
Great video!
```

the application could send:

```http
POST /comments
```

with:

```json
{
  "text": "Great video!"
}
```

The server receives this data and stores it.

# What Is JSON?

Most APIs use a format called **JSON**.

JSON stands for:

> JavaScript Object Notation

Despite its name, JSON is used by almost every programming language.

Python, Java, Go, Rust, C#, JavaScript, and many others support JSON.

---

# Why Do APIs Use JSON?

JSON is popular because it is:

- Human-readable
- Lightweight
- Easy to parse
- Language-independent

For example:

```json
{
  "name": "Alice",
  "age": 20
}
```

Even without programming knowledge, most people can understand this.

---

# JSON Objects

The most common JSON structure is an object.

Example:

```json
{
  "title": "Learn FastAPI",
  "checked": false,
  "priority": "medium"
}
```

This contains three pieces of information:

| Key      | Value         |
| -------- | ------------- |
| title    | Learn FastAPI |
| checked  | false         |
| priority | medium        |

---

# JSON Arrays

JSON can also store lists.

Example:

```json
[
  {
    "id": 1,
    "title": "Learn FastAPI"
  },
  {
    "id": 2,
    "title": "Buy groceries"
  }
]
```

This represents multiple todos.

---

# JSON vs Python Dictionaries

JSON objects look very similar to Python dictionaries.

---

## Python Dictionary

```python
{
    "title": "Learn FastAPI",
    "checked": False,
    "priority": "medium"
}
```

---

## JSON Object

```json
{
  "title": "Learn FastAPI",
  "checked": false,
  "priority": "medium"
}
```

Notice the difference:

---

## Python

```python
False
True
None
```

---

## JSON

```json
false
true
null
```

Everything else looks almost identical.

This is one reason Python works so well with APIs.

---

# Path Parameters vs Query Parameters vs Request Bodies

Beginners often confuse these concepts.

Let's understand the difference.

---

# Path Parameters

Path parameters are part of the URL itself.

Example:

```http
GET /todos/5
```

Here:

```text
5
```

is the path parameter.

It tells us:

> Fetch the todo with ID 5.

---

# FastAPI Example

```python
@app.get("/todos/{id}")
def get_todo(id):
    return id
```

FastAPI automatically extracts:

```text
5
```

from:

```text
/todos/5
```

---

# Query Parameters

Query parameters come after a question mark.

Example:

```http
GET /todos?priority=high
```

Here:

```text
priority=high
```

is a query parameter.

Query parameters are usually used for:

- Filtering
- Searching
- Sorting
- Pagination

---

# Examples

Filter by priority:

```http
GET /todos?priority=high
```

---

Filter completed tasks:

```http
GET /todos?checked=true
```

---

# Request Bodies

Request bodies contain larger pieces of data.

Example:

```http
POST /todos
```

with:

```json
{
  "title": "Learn FastAPI",
  "checked": false,
  "priority": "medium"
}
```

The server uses this information to create a new todo.

---

# Visual Comparison

---

## Path Parameter

```text
GET /todos/5
           ↑
         Path Parameter
```

---

## Query Parameter

```text
GET /todos?priority=high
             ↑
       Query Parameter
```

---

## Request Body

```text
POST /todos

{
    "title": "Learn FastAPI",
    "checked": false,
    "priority": "medium"
}
```

Everything inside the JSON is the request body.

---

# Why Not Put Everything in the URL?

Imagine this:

```text
POST /todos/LearnFastAPI/false/medium
```

This quickly becomes messy.

Instead, JSON provides a clean structure:

```json
{
  "title": "Learn FastAPI",
  "checked": false,
  "priority": "medium"
}
```

This is much easier to understand.

---

# How FastAPI Handles JSON

Suppose we write:

```python
@app.post("/todos")
def create_todo(todo: dict):
    return todo
```

FastAPI automatically:

---

## Step 1

Receives JSON:

```json
{
  "title": "Learn FastAPI",
  "checked": false,
  "priority": "medium"
}
```

---

## Step 2

Converts it into a Python dictionary:

```python
{
    "title": "Learn FastAPI",
    "checked": False,
    "priority": "medium"
}
```

---

## Step 3

Passes it to:

```python
create_todo(todo)
```

---

## Step 4

The function can work with it normally:

```python
todo["title"]
todo["priority"]
```

---

# Request and Response Cycle

The complete process looks like this:

```text
Client
   │
   │ POST /todos
   │
   │ {
   │   "title": "Learn FastAPI",
   │   "checked": false,
   │   "priority": "medium"
   │ }
   ▼
Server
   │
   │ Creates Todo
   ▼
Response
   │
   │ {
   │   "message": "Todo created"
   │ }
   ▼
Client
```

This cycle happens every time an API request is made.

# Understanding Each Field

---

## id

```json
"id": 1
```

A unique identifier for the todo.

No two todos should have the same ID.

---

## title

```json
"title": "Learn FastAPI"
```

The actual task.

---

## checked

```json
"checked": false
```

Indicates whether the task is completed.

---

## priority

```json
"priority": "medium"
```

Possible values:

```text
low
medium
high
```

This tells us how important the task is.
