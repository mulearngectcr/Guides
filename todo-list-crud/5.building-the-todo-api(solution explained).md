# 05 - Building the Todo API

> Day 3 – Building Your First CRUD API

In this guide, we will build our first complete CRUD application using FastAPI.

By the end, our API will support:

```http
GET    /todos
GET    /todos/{id}
POST   /todos
PUT    /todos/{id}
DELETE /todos/{id}
```

We will store everything in memory using a Python list.

No databases yet.

---

# Step 1: Creating main.py

Create a file called:

```text
main.py
```

Inside it, write:

```python
from fastapi import FastAPI

app = FastAPI()
```

---

# Understanding the Code

---

## Importing FastAPI

```python
from fastapi import FastAPI
```

This imports the `FastAPI` class.

We need this class to create our application.

Without this line, Python would not know what `FastAPI` means.

---

## Creating the App Object

```python
app = FastAPI()
```

This creates our FastAPI application.

Think of `app` as the main controller of our API.

All routes will be attached to this object.

Example:

```python
@app.get("/")
```

The `get()` function belongs to the `app` object.

---

# Step 2: Creating Our Temporary Database

Add:

```python
todos = []
```

Your file now becomes:

```python
from fastapi import FastAPI

todos = []
```

---

# Why Are We Using a List?

Normally, applications use databases.

Examples:

- MySQL
- PostgreSQL
- MongoDB

But databases add extra complexity.

For our first CRUD project, a simple Python list is enough.

---

# How Will Todos Be Stored?

Eventually, our list will look like this:

```python
todos = [
    {
        "id": 1,
        "title": "Learn FastAPI",
        "checked": False,
        "priority": "medium"
    },

    {
        "id": 2,
        "title": "Buy groceries",
        "checked": False,
        "priority": "high"
    }
]
```

Each item in the list is a dictionary.

Each dictionary represents one todo.

# Step 3: Creating GET /todos

Add:

```python
@app.get("/todos")
```

# Understanding @app.get()

```python
@app.get("/todos")
```

This is called a decorator.

It tells FastAPI:

> When someone visits `/todos` using a GET request, execute the function below.

# Understanding the Function

```python
def get_todos():
```

This function runs whenever:

```http
GET /todos
```

is called.


# Step 5: Creating GET /todos/{id}

Now we want to fetch a single todo.

Add:

```python
@app.get("/todos/{id}")
def get_todo(id: int):

    for todo in todos:

        if todo["id"] == id:
            return todo
```

---

# Understanding Path Parameters

```python
"/todos/{id}"
```

The `{id}` part is called a path parameter.

Examples:

```http
GET /todos/1
GET /todos/2
GET /todos/3
```

FastAPI automatically extracts the number.

---

# Why id: int?

```python
id: int
```

This tells FastAPI:

> Convert the value into an integer.

So:

```text
"/todos/1"
```

becomes:

```python
1
```

instead of:

```python
"1"
```

---

Example:

```text

Todo 3
```

until it finds a match.

# Checking the ID

# Step 6: Creating PUT /todos/{id}

We now want to update existing todos.

Add:

```python
@app.put("/todos/{id}")
def update_todo(id: int, updated_todo: dict):

    for todo in todos:

        if todo["id"] == id:

            todo["title"] = updated_todo["title"]
            todo["checked"] = updated_todo["checked"]
            todo["priority"] = updated_todo["priority"]

            return {
                "message": "Todo updated successfully"
            }
```

---

# Understanding updated_todo

```python
updated_todo: dict
```

The user sends new data.

Example:

```json
{
  "title": "Master FastAPI",
  "checked": true,
  "priority": "high"
}
```

---

# Updating the Values

We modify the existing dictionary.

---

Title:

```python

```

---

Checked:

```python

```

---

Priority:

```python

```

# Why Does This Work?

Dictionaries are mutable.

Mutable means:

> They can be changed after creation.

The original object is modified directly.

# Step 7: Creating DELETE /todos/{id}

Finally, we want to remove todos.

Add:

```python
@app.delete("/todos/{id}")
def delete_todo(id: int):

    for todo in todos:

        if todo["id"] == id:

            todos.remove(todo)

            return {
                "message": "Todo deleted successfully"
            }
```

---

# Understanding remove()

```python
todos.remove(todo)
```

This removes the item from the list.

---

```text
]
```

---

After deleting Todo 2:

```text

    Todo 1,
    Todo 3
```

The list automatically adjusts itself.

# What Happens If We Restart?

Remember:

```python
todos = []
```

lives only in memory.

If you stop:

```bash
CTRL + C
```

and restart:

```bash
uvicorn main:app --reload
```

all todos disappear.

That is completely normal.
