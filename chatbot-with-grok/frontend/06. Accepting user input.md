# Frontend Guide

# Accepting User Input

Up until now, our chatbot can display previous messages.

However,

the user still has no way to send a new message.

We now need an input box where users can type their questions.

Streamlit provides a component specifically designed for chat applications.

```python
st.chat_input()
```

This creates an input box fixed at the bottom of the page, similar to applications like ChatGPT, WhatsApp, or Discord.

---

# Our Code

We write

```python
prompt = st.chat_input("Ask something...")
```

Although this is only one line of code, there's a lot happening behind the scenes.

Let's understand it carefully.

---

# What Does `st.chat_input()` Do?

When Streamlit reaches

```python
st.chat_input("Ask something...")
```

it displays a chat input box.

It looks something like this.

```
+----------------------------------------+
| Ask something...                  [➤]  |
+----------------------------------------+
```

The text

```
Ask something...
```

is called the **placeholder**.

It is simply a hint shown before the user starts typing.

---

# Waiting for the User

Unlike most Streamlit widgets,

`st.chat_input()` behaves a little differently.

It waits until the user submits a message.

Suppose the page loads.

The user hasn't typed anything yet.

In that case,

```python
prompt = st.chat_input("Ask something...")
```

returns

```python
None
```

because no message has been submitted.

---

# User Types a Message

Now suppose the user enters

```
What is FastAPI?
```

and presses Enter.

This time,

the same line becomes

```python
prompt = "What is FastAPI?"
```

Instead of returning `None`,

it returns the text entered by the user.

---

# Understanding the Variable

We usually store the result inside a variable.

```python
prompt = st.chat_input("Ask something...")
```

The variable

```python
prompt
```

will contain one of two values.

If the user hasn't submitted anything,

```python
prompt
```

contains

```python
None
```

If the user submits a message,

```python
prompt
```

contains the text they entered.

For example,

```python
"What is Python?"
```

---

# Why Does It Return `None`?

Imagine it always returned an empty string.

Then we couldn't distinguish between

- the user hasn't typed anything yet, and
- the user intentionally submitted an empty message.

Returning `None` clearly tells us that no submission has happened.

---

# Checking Whether the User Sent a Message

Since `prompt` may be `None`,

we usually write

```python
if prompt:
```

This condition is true only when the user has actually submitted a message.

If nothing has been entered,

the block is skipped.

---

# Visualizing the Flow

```

      │

      ▼

      │

      ▼

      │

 ┌────┴────┐
 │         │
No        Yes
 │         │
 ▼         ▼

```

---

# Why Store It in a Variable?

Suppose the user types

```
Hello
```

Now

```python
prompt
```

contains

```python
"Hello"
```

We can use this variable throughout the rest of our program.

For example,

- display the user's message,
- send it to the FastAPI backend,
- save it in Session State,
- or include it in future API requests.

Everything starts with this single variable.

---

# Recap

The code is simply

```python
prompt = st.chat_input("Ask something...")
```

Here's what happens.

- Streamlit displays a chat input box.
- The function waits for the user to submit a message.
- Before submission, it returns `None`.
- After submission, it returns the user's text.
- We store that value inside the `prompt` variable for later use.

In the next section, we'll use this `prompt` variable to add the user's message to our conversation history and immediately display it in the chat window.
