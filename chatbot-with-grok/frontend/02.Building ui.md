# Frontend Guide

# Building the Chat Interface

Our frontend is now configured and ready.

The next step is creating the chat interface.

A chatbot should be able to:

- display previous messages,
- accept new user input,
- and show the user's message immediately after it's is sent.

We'll build all of that in this section.

---

# Displaying Previous Messages

The first piece of code is

```python
for message in st.session_state.messages:
    with st.chat_message(message["role"]):
        st.markdown(message["content"])
```

Remember,

in the previous section, we created a list named `messages` inside `st.session_state`.

Every conversation in our chatbot will be stored inside this list.

For example,

```python
[
    {
        "role": "user",
        "content": "Hello"
    },
    {
        "role": "assistant",
        "content": "Hi! How can I help you?"
    }
]
```

The `for` loop iterates through every message stored in this list.

For each message,

```python
with st.chat_message(message["role"]):
```

creates the appropriate chat bubble.

If the role is `"user"`,

Streamlit displays a user message.

If the role is `"assistant"`,

it displays an assistant message.

Finally,

```python
st.markdown(message["content"])
```

renders the actual text inside that bubble.

Since this loop runs every time Streamlit reruns the script, the entire conversation is reconstructed automatically from `st.session_state.messages`.

---

# Accepting User Input

Next, we add an input box.

```python
if prompt := st.chat_input("Type your message..."):
```

Unlike a normal text input,

`st.chat_input()` is designed specifically for chat applications.

It displays an input box fixed at the bottom of the page.

When the user presses **Enter**,

the function returns the text they entered.

For example,

```text
What is FastAPI?
```

If the user hasn't submitted anything yet,

it returns `None`.

---

# Understanding the Walrus Operator

You may notice an unfamiliar operator.

```python
:=
```

This is called the **walrus operator**.

Normally, we would write

```python
prompt = st.chat_input("Type your message...")

if prompt:
```

The walrus operator combines these two steps into a single line.

It first stores the result inside `prompt`, and then immediately checks whether that value is truthy.

Since `None` is considered false,

the code inside the `if` block only executes when the user actually submits a message.

---

# Saving the User's Message

Once the user submits a prompt,

the first thing we do is save it.

```python
st.session_state.messages.append({
    "role": "user",
    "content": prompt
})
```

Here, we're adding a new dictionary to our conversation history.

For example, if the user types

```text
Explain FastAPI.
```

the following dictionary is appended.

```python
{
    "role": "user",
    "content": "Explain FastAPI."
}
```

From now on,

this message becomes part of the conversation history and will be displayed whenever Streamlit reruns the application.

---

# Displaying the User's Message

After saving it,

we immediately display it on the screen.

```python
with st.chat_message("user"):
    st.markdown(prompt)
```

This creates a user chat bubble and renders the message inside it.

Although the message has already been stored in Session State,

displaying it immediately provides instant feedback to the user.

Otherwise,

they would have to wait until the entire script reruns before seeing their own message.

This makes the application feel much more responsive.

---

# Request Flow So Far

At this point,

our application follows this sequence.

```
User Types a Message

        │

        ▼

st.chat_input()

        │

        ▼

Store the Message

(Session State)

        │

        ▼

Display the User Bubble

        │

        ▼

Wait for Assistant Response
```

Notice that we haven't contacted the backend yet.

We've only handled the frontend portion of the interaction.

The next step is sending the user's prompt to our FastAPI backend and displaying the AI's response.

---

# Recap

In this section, we:

- rebuilt previous conversations from Session State,
- added a chat input box,
- used the walrus operator to simplify input handling,
- stored the user's message in Session State,
- and displayed it immediately inside the chat interface.

At this point, our chatbot behaves like a real messaging application.

The only missing piece is connecting it to the FastAPI backend to generate responses.
