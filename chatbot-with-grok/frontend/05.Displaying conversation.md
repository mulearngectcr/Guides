# Frontend Guide
## Displaying the Conversation

In the previous section, we created a place to store our conversation.

```python
st.session_state.messages
```

Now we need to display those messages on the screen.

Remember, every time Streamlit reruns the script, the webpage is built again from top to bottom.

If we don't redraw the old messages, the chat window would appear empty after every interaction.

This section explains how Streamlit recreates the conversation using the messages stored in Session State.

---

# Displaying Previous Messages

Our code looks like this.

```python
for message in st.session_state.messages:
    with st.chat_message(message["role"]):
        st.markdown(message["content"])
```

Although it's only three lines long, it rebuilds the entire chat history every time the page is refreshed.

Let's understand it one line at a time.

---

# Understanding the for Loop

The first line is

```python
for message in st.session_state.messages:
```

Earlier, we created

```python
st.session_state.messages
```

which stores a list of dictionaries.

Example:

```python
[
    {"role":"user","content":"Hello"},
    {"role":"assistant","content":"Hi!"},
    {"role":"user","content":"What is Python?"}
]
```

The `for` loop goes through this list one message at a time.

During each iteration, the variable `message` refers to one dictionary from the list until every stored message has been processed.

---

# Why Do We Loop?

Whether there are 2 messages or 2,000 messages, the exact same loop displays all of them automatically. We don't have to manually create a chat bubble for every single message.

---

# Creating a Chat Bubble

```python
with st.chat_message(message["role"]):
```

This creates a temporary chat message container.

Everything inside this block belongs to one chat bubble.

The value of `message["role"]` decides whether Streamlit renders a **user** bubble or an **assistant** bubble.

---

# Understanding `with`

`with` creates a **context manager**.

Think of it as:

```
Open Chat Bubble
      ↓
Display Content
      ↓
Close Chat Bubble
```

Everything indented beneath the `with` statement belongs inside that chat bubble.

---

# Displaying the Message

Inside the bubble we write

```python
st.markdown(message["content"])
```

Suppose the current message is

```python
{
    "role": "assistant",
    "content": "Python is a programming language."
}
```

The bubble displays

> Python is a programming language.

---

# Why Use Markdown?

Instead of `st.write()`, we use `st.markdown()` because AI responses often contain Markdown formatting such as headings, bold text, lists and code blocks.

For example, if the AI returns

````text
## Python

Python is a programming language.

```python
print("Hello")
```
````

Streamlit renders it with proper formatting instead of displaying the raw Markdown symbols.

---

# Following One Iteration

Suppose

```python
message = {
    "role": "assistant",
    "content": "Hello! How can I help you?"
}
```

The loop performs the following steps:

1. Picks the dictionary from `st.session_state.messages`.
2. Creates an assistant chat bubble.
3. Displays `Hello! How can I help you?` inside that bubble.
4. Moves to the next dictionary.

This repeats until every message has been displayed.

---

# What Happens During Every Rerun?

Imagine the conversation is

```
User: Hello
Assistant: Hi!
User: What is FastAPI?
Assistant: FastAPI is a Python web framework.
```

When the user sends another prompt, Streamlit reruns the entire script.

Without this loop, only the newest message would appear.

Instead, the loop redraws every previous message from `st.session_state.messages`, rebuilding the complete conversation before the rest of the application continues executing.

To the user, it feels like the conversation was never lost.

---

# Visualizing the Process

```
Session State

messages
│
├── User : Hello
├── Assistant : Hi!
├── User : What is FastAPI?
└── Assistant : FastAPI is a Python web framework.

        │
        ▼
for message in messages
        │
        ▼
Create Chat Bubble
        │
        ▼
Render Markdown
        │
        ▼
Next Message
        │
        ▼
Repeat
```

---

# Why Store Messages?

Without Session State, Python forgets every previous message whenever Streamlit reruns the script.

`st.session_state.messages` acts as the application's memory, allowing us to rebuild the conversation on every execution.

---

# Recap

```python
for message in st.session_state.messages:
    with st.chat_message(message["role"]):
        st.markdown(message["content"])
```

- The `for` loop iterates through every stored message.
- `st.chat_message()` creates the correct chat bubble.
- `st.markdown()` renders the message while preserving Markdown formatting.
- The loop runs on every rerun, recreating the complete conversation from Session State.

Without this loop, the chat window would appear empty after every interaction.
