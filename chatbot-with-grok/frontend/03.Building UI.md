# The Complete Code

We'll build the following application.

```python
import os

import requests
import streamlit as st
from dotenv import load_dotenv

load_dotenv()

API_URL = os.getenv(
    "API_URL",
    "http://127.0.0.1:8000/chat"
)

st.set_page_config(
    page_title="AI Chatbot",
    page_icon="🤖"
)

st.title("🤖 AI Chatbot")

if "messages" not in st.session_state:
    st.session_state.messages = []

for message in st.session_state.messages:
    with st.chat_message(message["role"]):
        st.markdown(message["content"])

if prompt := st.chat_input("Type your message..."):

    st.session_state.messages.append(
        {
            "role": "user",
            "content": prompt,
        }
    )

    with st.chat_message("user"):
        st.markdown(prompt)

    with st.chat_message("assistant"):

        with st.spinner("Thinking..."):

            try:

                response = requests.post(
                    API_URL,
                    json={
                        "message": prompt
                    },
                    timeout=60
                )

                if response.status_code == 200:
                    answer = response.json()["response"]
                else:
                    answer = f"Error: {response.text}"

            except Exception as e:
                answer = str(e)

            st.markdown(answer)

    st.session_state.messages.append(
        {
            "role": "assistant",
            "content": answer,
        }
    )
```

Don't worry if this looks overwhelming.

We'll understand it section by section.

# Importing Libraries

The first few lines import the libraries we'll use.

```python
import os

import requests
import streamlit as st
from dotenv import load_dotenv
```

Let's understand each one.

---

## Importing Streamlit

```python
import streamlit as st
```

This imports the Streamlit framework.

Notice that we rename it to

```python
st
```

instead of writing

```python
streamlit.title(...)
```

every time.

Now we can simply write

```python
st.title(...)
```

Almost every Streamlit project follows this convention.

---

## Importing Requests

```python
import requests
```

Our frontend needs to communicate with FastAPI.

The `requests` library allows us to send HTTP requests.

Later, we'll use

```python
requests.post(...)
```

to send the user's message to the backend.

---

## Importing os

```python
import os
```

The `os` module allows Python to interact with the operating system.

In our application,

we'll use it to read environment variables.

---

## Importing dotenv

```python
from dotenv import load_dotenv
```

Earlier,

we created a `.env` file.

This function loads the variables stored inside that file into our Python program.

Without calling

```python
load_dotenv()
```

Python wouldn't know about the variables defined in `.env`.

---

# Loading Environment Variables

Immediately after importing,

we write

```python
load_dotenv()
```

This tells Python

> "Read the `.env` file and make its variables available to the program."

Suppose our `.env` file contains

```
API_URL=http://127.0.0.1:8000/chat
```

After calling

```python
load_dotenv()
```

we can access it using

```python
os.getenv(...)
```

---

# Reading the Backend URL

Next,

we define

```python
API_URL = os.getenv(
    "API_URL",
    "http://127.0.0.1:8000/chat"
)
```

Let's understand this carefully.

---

## os.getenv()

This function reads an environment variable.

The first argument is the variable's name.

```python
"API_URL"
```

Python searches for this variable inside the environment.

---

## The Second Argument

Notice that we passed another value.

```python
"http://127.0.0.1:8000/chat"
```

This is called the **default value**.

If `API_URL` isn't found,

Python simply uses this value instead.

This makes our application more robust.

Even if someone forgets to create a `.env` file,

the frontend can still connect to a locally running backend.

# Understanding Streamlit's Execution Model

Before writing the rest of our application,

it's important to understand how Streamlit works.

Unlike traditional web frameworks,

Streamlit executes your entire Python script **from top to bottom every time the user interacts with the page**.

For example,

suppose the user

- clicks a button,
- submits a message,
- changes a slider.

Instead of executing only a small part of the program,

Streamlit reruns the entire script.

At first,

this sounds inefficient.

However,

it greatly simplifies application development.

The challenge is remembering information between reruns.

That's where **Session State** comes in.
