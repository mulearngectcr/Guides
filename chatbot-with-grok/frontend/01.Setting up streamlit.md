# Frontend Guide

# Part 1 - Setting Up the Streamlit Frontend

In the previous part of this project, we built our chatbot's backend using FastAPI.

The backend exposes an API that accepts a user's message and returns the AI's response.

However, APIs aren't meant to be used directly by end users.

We need a frontend where users can:

- type their messages,
- view previous conversations,
- and read the assistant's responses.

For this project, we'll build that frontend using **Streamlit**.

---

# Why Streamlit?

There are many ways to build a frontend.

For example,

- HTML, CSS and JavaScript
- React
- Vue
- Angular

However, all of these require learning additional frontend technologies.

Since our focus is learning **FastAPI and AI application development**, we'll use Streamlit.

Streamlit lets us build interactive web applications entirely in Python.

It also provides built-in components for chat interfaces, making it an excellent choice for AI projects and prototypes.

---

# Our Frontend

Create a new file named

```
frontend.py
```

Our application starts with the following code.

```python
import os
import streamlit as st
import requests
from dotenv import load_dotenv
```

Let's understand why each import is required.

---

# Importing `os`

```python
import os
```

The `os` module allows us to read environment variables.

Instead of hardcoding our backend URL inside the code, we'll store it in a `.env` file.

This makes the application easier to configure when deploying or changing environments.

---

# Importing Streamlit

```python
import streamlit as st
```

This imports the Streamlit library.

Almost every frontend component we'll create—titles, chat messages, input boxes, loading spinners, and session state—comes from this module.

We import it as `st` because that's the standard convention used in Streamlit applications.

---

# Importing Requests

```python
import requests
```

Our frontend needs to communicate with the FastAPI backend.

The `requests` library allows us to send HTTP requests to our API.

Later in this guide, we'll use it to send the user's prompt to the backend and receive the AI's response.

---

# Importing `load_dotenv`

```python
from dotenv import load_dotenv
```

The `python-dotenv` package loads variables stored inside a `.env` file into our application's environment.

This lets us keep configuration values outside our source code.

---

# Loading Environment Variables

```python
load_dotenv()
```

This reads the `.env` file and loads its contents into the environment.

For example, suppose our `.env` file contains

```env
API_URL=http://127.0.0.1:8000/chat
```

After calling `load_dotenv()`, we can access this value using the `os` module.

---

# Reading the Backend URL

```python
API_URL = os.getenv(
    "API_URL",
    "http://127.0.0.1:8000/chat"
)
```

Here, we read the value of the `API_URL` environment variable.

If the variable exists inside the `.env` file, that value is used.

Otherwise, the second argument acts as a default.

```python
"http://127.0.0.1:8000/chat"
```

This is the endpoint exposed by the FastAPI backend that we built earlier.

Whenever the user sends a message, our frontend will send a POST request to this URL.

---

# Configuring the Page

Next, we configure some basic properties of our application.

```python
st.set_page_config(
    page_title="AI Chatbot",
    page_icon="🤖"
)
```

This function customizes the browser tab.

Here,

- `page_title` sets the title shown in the browser tab.
- `page_icon` sets the favicon displayed next to the title.

These settings only affect the appearance of the page and don't influence the chatbot's functionality.

---

# Adding a Title

```python
st.title("🤖 AI Chatbot")
```

This displays a heading at the top of the application.

When the app runs, users will see

```
🤖 AI Chatbot
```

at the top of the page.

---

# Initializing Session State

Finally, we initialize our chat history.

```python
if "messages" not in st.session_state:
    st.session_state.messages = []
```

Unlike traditional web frameworks, Streamlit reruns the entire script whenever the user interacts with the page.

If we stored our conversation in a normal Python variable, it would disappear after every interaction.

`st.session_state` provides a way to persist data across these reruns.

Here, we create a key named `messages` if it doesn't already exist.

Initially, it contains an empty list.

```python
[]
```

As users interact with the chatbot, we'll store each message inside this list.

This allows us to rebuild the conversation every time the application reruns.
