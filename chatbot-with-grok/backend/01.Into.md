## 📖 Introduction

Over the past few days, you've explored the fundamentals of building REST APIs using FastAPI. You created endpoints, worked with request bodies, validated incoming data using Pydantic, and learned how a backend communicates with clients.

In this task, we're going one step further.

Instead of returning static responses, we'll build an **AI-powered chatbot** capable of answering user queries using a Large Language Model (LLM).

By the end of this task, you'll have built and deployed a complete AI application consisting of:

- A FastAPI backend
- A Streamlit frontend
- Integration with Groq's LLM API
- Persistent chat memory
- Cloud deployment

This is the first complete AI application you'll build from scratch.

# 📂 Project Structure

Our project looks like this:

```
chatbot/
│
├── backend.py
├── frontend.py
├── requirements.txt
├── .env
├── memory.json
└── .gitignore
```

Every file has a specific responsibility.

| File | Purpose |
|-------|----------|
| backend.py | FastAPI backend that communicates with Groq |
| frontend.py | Streamlit user interface |
| memory.json | Stores conversation history |
| .env | Stores secret API keys |
| requirements.txt | Project dependencies |
| .gitignore | Prevents sensitive files from being uploaded to GitHub |

# 🏗 Overall Architecture

```
                 User
                   │
                   ▼
          Streamlit Frontend
                   │
        HTTP POST Request
                   │
                   ▼
            FastAPI Backend
                   │
      Load Previous Messages
                   │
                   ▼
              Groq API
                   │
         AI Generates Response
                   │
                   ▼
      Save Conversation History
                   │
                   ▼
         Return Response to UI
```

Notice that the frontend **never talks directly to the AI model.**

Instead,

Frontend → Backend → Groq API

This is the architecture followed by almost every production AI application.
