# 02 - Setting Up Rate Limiting

> Day 7 – Rate Limiting

---

# Introduction

We will use a library called `slowapi` to add rate limiting to our FastAPI application.

slowapi is built specifically for FastAPI and is very straightforward to set up.

---

# Step 1: Install slowapi

```bash
pip install slowapi
```

---

# Step 2: Imports

Add these imports to your `main.py`:

```python
from fastapi import Request
from slowapi import Limiter, _rate_limit_exceeded_handler
from slowapi.util import get_remote_address
from slowapi.errors import RateLimitExceeded
```

---

# Understanding the Imports

---

## Limiter

```python
from slowapi import Limiter
```

This is the main rate limiting class.

We use it to define our limits.

---

## _rate_limit_exceeded_handler

```python
from slowapi import _rate_limit_exceeded_handler
```

This is a built-in handler that slowapi provides.

When a rate limit is exceeded, this function automatically returns a proper 429 response.

We do not need to write this logic ourselves.

---

## get_remote_address

```python
from slowapi.util import get_remote_address
```

This tells slowapi how to identify each client.

`get_remote_address` uses the client's IP address.

So each IP address gets its own separate request counter.

---

## RateLimitExceeded

```python
from slowapi.errors import RateLimitExceeded
```

This is the error that slowapi raises when a limit is exceeded.

We use it to attach our handler.

---

## Request

```python
from fastapi import Request
```

slowapi needs access to the raw HTTP request to read the client's IP address.

We will need this for endpoints with a per-endpoint limit override.

---

# Step 3: Create the Limiter

```python
limiter = Limiter(
    key_func=get_remote_address,
    default_limits=["10/minute"]
)
```

---

# Understanding This

---

## key_func=get_remote_address

This tells slowapi:

> Identify each client by their IP address.

Each IP address gets its own counter.

---

## default_limits=["10/minute"]

This sets a global limit of 10 requests per minute on every endpoint.

You do not need to add anything to individual endpoints for this to work.

---

# Step 4: Attach the Limiter to the App

```python
app = FastAPI()
app.state.limiter = limiter
app.add_exception_handler(RateLimitExceeded, _rate_limit_exceeded_handler)
```

---

# Understanding This

---

## app.state.limiter = limiter

This attaches our limiter to the FastAPI application.

slowapi looks for the limiter here automatically.

---

## app.add_exception_handler(RateLimitExceeded, _rate_limit_exceeded_handler)

This tells FastAPI:

> When a RateLimitExceeded error occurs, use this handler to return a 429 response.

Without this line, FastAPI would return a 500 Internal Server Error instead of a proper 429.

---

# Step 5: Global Limit in Action

With just these lines, every endpoint is now rate limited:

```python
@app.get("/todos", dependencies=[Depends(authenticate)])
def get_todos():
    return todos
```

No extra code needed on the endpoint itself.

The global limit of 10/minute applies automatically.

---

# Step 6: Overriding the Global Limit on POST /todos

Write operations should be stricter.

We override the global limit on `POST /todos`:

```python
@app.post("/todos", status_code=201, dependencies=[Depends(authenticate)])
@limiter.limit("3/minute")
async def create_todo(request: Request, todo: Todo):
    global id_counter
    new_todo = todo.model_dump()
    new_todo["id"] = id_counter
    todos.append(new_todo)
    id_counter += 1
    return {
        "message": "Todo added successfully"
    }
```

---

# Understanding the Override

---

## @limiter.limit("3/minute")

This decorator overrides the global limit for this specific endpoint.

`POST /todos` now allows only 3 requests per minute instead of 10.

---

## async def

```python
async def create_todo(request: Request, todo: Todo):
```

When using `@limiter.limit()` on a specific endpoint, slowapi requires:

- The function to be `async def`
- `request: Request` to be the first parameter

slowapi uses the request object to read the client's IP address for this specific endpoint.

---

## request: Request

```python
async def create_todo(request: Request, todo: Todo):
```

You do not need to use `request` yourself.

slowapi reads it automatically behind the scenes.

Just make sure it is the first parameter.

---

# Why Only POST Needs async and Request?

The global limiter handles all other endpoints automatically without needing `request: Request`.

Only endpoints with a `@limiter.limit()` decorator need:

- `async def`
- `request: Request` as first parameter

So all other endpoints stay clean:

```python
@app.get("/todos", dependencies=[Depends(authenticate)])
def get_todos():
    return todos
```

---

# What Happens When the Limit Is Exceeded?

slowapi automatically returns:

```json
{
  "error": "Rate limit exceeded: 3 per 1 minute"
}
```

with status code:

```text
429 Too Many Requests
```

We did not write any of this.

`_rate_limit_exceeded_handler` handles it for us.
