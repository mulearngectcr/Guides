# 03 - Testing Rate Limiting

> Day 7 – Testing Rate Limits

---

# Introduction

Now that rate limiting is set up, let's test it.

We want to confirm:

- Normal requests work as expected
- Requests beyond the limit return 429

---

# Starting the Server

```bash
uvicorn main:app --reload
```

---

# Opening Swagger UI

```text
http://127.0.0.1:8000/docs
```

Enter your API key using the **Authorize** button before testing.

---

# Testing the Global Limit on GET /todos

`GET /todos` has the global limit of 10 requests per minute.

Click:

```text
GET /todos → Try it out → Execute
```

Repeat this more than 10 times within one minute.

---

# Expected Response After Exceeding the Limit

```json
{
  "error": "Rate limit exceeded: 10 per 1 minute"
}
```

With status code:

```text
429 Too Many Requests
```

---

# Testing the Stricter Limit on POST /todos

`POST /todos` has a stricter limit of 3 requests per minute.

Click:

```text
POST /todos → Try it out
```

Enter:

```json
{
  "title": "Test Todo",
  "checked": false,
  "priority": "medium"
}
```

Click **Execute** more than 3 times within one minute.

---

# Expected Response After Exceeding the Limit

```json
{
  "error": "Rate limit exceeded: 3 per 1 minute"
}
```

With status code:

```text
429 Too Many Requests
```

---

# Waiting for the Window to Reset

After hitting the limit, wait for one minute.

Then try again.

The request should succeed normally.

This confirms the time window reset correctly.

---

# Visualizing the Flow

```text
Request comes in
│
├── Auth check
│     ├── Invalid key → 401
│     └── Valid key → continue
│
├── Rate limit check
│     ├── Limit exceeded → 429
│     └── Within limit → continue
│
└── Endpoint runs → 200 / 201
```

Auth runs first, then rate limiting, then the endpoint.

---

# Complete Testing Checklist

```text
✅ GET /todos with correct key → 200
✅ GET /todos 11+ times in a minute → 429
✅ POST /todos with correct key → 201
✅ POST /todos 4+ times in a minute → 429
✅ Wait one minute → requests work again
✅ All other endpoints still work correctly
```
