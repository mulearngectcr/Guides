# Part 1 — Understanding the Workflow

Before building anything, let's understand what is happening.

---

# What is a Workflow?

A workflow is a sequence of connected steps.

Example:

```text
Wake Up
 ↓
Brush Teeth
 ↓
Eat Breakfast
 ↓
Go To College
```

Each step depends on the previous step.

n8n works exactly the same way.

---

# What is a Node?

A node is a single step in a workflow.

Examples:

- Send Email
- Fetch Data
- Create File
- Save Database Record
- Send WhatsApp Message

Each action is represented as a node.

---

# Nodes Used Today

## 1. Manual Trigger

Starts the workflow manually.

Think of it like pressing the Start button.

```text
Manual Trigger
```

---

## 2. HTTP Request

Used to communicate with APIs.

Think of it as:

```text
"Hey API, give me some data."
```

The API responds with data.

---

## 3. Send Email

Takes the fetched joke and emails it.

---

# Understanding APIs

API stands for:

```text
Application Programming Interface
```

APIs allow applications to talk to each other.

---

# Example

You:

```text
Give me a joke
```

API:

```json
{
  "setup": "Why did the scarecrow win an award?",
  "punchline": "Because he was outstanding in his field!"
}
```

---

# What Happens Internally?

Step 1:

```text
Manual Trigger
```

starts workflow.

Step 2:

```text
HTTP Request
```

fetches joke.

Step 3:

```text
Send Email
```

uses the fetched joke.

---

# Data Flow

```text
Manual Trigger
      ↓
HTTP Request
      ↓
{
 setup: "...",
 punchline: "..."
}
      ↓
Send Email
```

The HTTP Request node passes data to the Email node.

This is called:

```text
Node Output
```

Understanding node output is one of the most important concepts in n8n.

---

# Viewing Node Output

After executing a node:

1. Click the node
2. Open Output tab
3. Inspect returned JSON

You should see something like:

```json
{
  "type": "general",
  "setup": "...",
  "punchline": "...",
  "id": 42
}
```

Always inspect outputs when debugging workflows.
