# Challenge 1 — Better Email Formatting

Instead of:

```text
Setup:
...
Punchline:
...
```

Try:

```text
😂 Random Joke

Setup:
{{$json.setup}}

Punchline:
{{$json.punchline}}

Generated At:
{{$now}}
```

Much more readable.

# Challenge 2 — Error Handling

Real APIs can fail.

Possible reasons:

- Network issues
- API downtime
- Invalid URL

---

# Using Error Branches

Enable:

```text
Continue On Fail
```

inside HTTP Request.

If the API fails:

- workflow should not crash
- error can be handled gracefully

---

# Example Error Email

Subject:

```text
Joke Generator Failed
```

Body:

```text
Unable to fetch joke.

Time:
{{$now}}
```

---

# Bonus 1 — IF Node

Add:

```text
HTTP Request
      ↓
      IF
```

Purpose:

Check joke length.

---

# Example Condition

```text
{{$json.setup.length}}
```

Greater than:

```text
20
```

If TRUE:

```text
Send Email
```

If FALSE:

```text
Fetch Another Joke
```

---

# Bonus 2 — Second HTTP Request

Workflow:

```text
Manual Trigger
      ↓
HTTP Request
      ↓
IF
   ↙     ↘
TRUE    FALSE
 ↓         ↓
Email   HTTP Request 2
            ↓
         Email
```

If joke is too short:

Fetch another one.

# Bonus 3 — Dynamic Subject

Add a Set Node.

Workflow:

```text
HTTP Request
      ↓
Set
      ↓
Email
```

Create field:

```text
subject
```

Value:

```text
Daily Joke - {{$now}}
```

Then in Email Subject:

```text
{{$json.subject}}
```
