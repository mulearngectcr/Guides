# Part 2 — Building The Workflow

Let's build the workflow step by step.

---

# Step 1 — Create Workflow

Click:

```text
New Workflow
```

Rename it:

```text
Random Joke Generator
```

---

# Step 2 — Add Manual Trigger

Click:

```text
+
```

Search:

```text
Manual Trigger
```

Add it.

This node requires no configuration.

---

# Step 3 — Add HTTP Request

Click the + icon.

Search:

```text
HTTP Request
```

Connect it to Manual Trigger.

---

# Configure HTTP Request

Method:

```text
GET
```

URL:

```text
https://official-joke-api.appspot.com/random_joke
```

---

# Test API

Click:

```text
Execute Node
```

Expected output:

```json
{
  "setup": "...",
  "punchline": "..."
}
```

If you receive JSON data, the API works.

---

# Step 4 — Add Email Node

Search:

```text
Send Email
```

Connect it.

---

# Configure Email Credentials

Depending on provider:

- Gmail
- SMTP
- Outlook

Enter credentials provided by your instructor.

---

# Email Subject

Example:

```text
Your Random Joke Has Arrived 😂
```

---

# Email Body

Switch to expression mode when necessary.

Example:

```text
Joke Setup:

{{$json.setup}}

Punchline:

{{$json.punchline}}
```

---

# Add Timestamp

n8n expression:

```text
{{ $now }}
```

Example:

```text
Joke Setup:
{{$json.setup}}

Punchline:
{{$json.punchline}}

Generated At:
{{$now}}
```

---

# Test Workflow

Click:

```text
Joke Setup:

```

---

# Verify Randomness

Run workflow multiple times.

Each run should return a different joke.

Document this in screenshots if required.

---

# Common Issues

## No Output

Check API URL.

---

## Email Not Sent

Verify:

- SMTP credentials
- App passwords
- Recipient email

---

## Empty Fields

Ensure:

```text
{{$json.setup}}
{{$json.punchline}}
```

are typed correctly.

---

# Debugging Tip

Always execute nodes individually.

Example:

```text
Manual Trigger
      ↓
HTTP Request
```

Verify output first before configuring email.
